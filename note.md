## Taste vs Accessibility

That's a really thoughtful distinction — accessibility isn't just WCAG compliance, it's whether *this type of person* can actually navigate *this UI* comfortably.

So the persona system covers the "can different people use it" dimension. What covers the "taste" dimension then — the aesthetic and UX quality judgment? Is that a separate layer, or do you see taste as emergent from aggregating enough persona perspectives?

Because there's an interesting design question there: taste might be more universal (good visual hierarchy, breathing room, clear affordances), while accessibility is more persona-specific. They're related but not the same thing.
